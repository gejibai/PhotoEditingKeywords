# 项目升级需求：日常修图关键词完善器 Web App

## 1. 项目目标

将现有的单文件 HTML「日常修图关键词完善器」升级为一个可维护的 Web App。

用户输入一句粗略想法后，系统可以：

1. 通过离线规则自动拆解字段；
2. 可选通过 AI 模型进一步理解和补全字段；
3. 生成可复制的：
   - 完整提示词
   - 关键词分层
   - 一句话压缩版
   - JSON 结构化提示词

项目用途是日常修图、生图、照片风格化提示词整理，不是生图工具，不上传和处理真实图片。

## 2. 技术栈

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Vercel 部署
- Next.js Route Handler 做后端 API
- API Key 通过环境变量保存
- API Key 禁止出现在前端代码里

Next.js App Router 的 Route Handler 可通过 `app/**/route.ts` 创建自定义请求处理逻辑，适合做 `/api/*` 后端接口。:contentReference[oaicite:0]{index=0}  
环境变量方面，Next.js 中非 `NEXT_PUBLIC_` 的变量默认只在服务端可用；如果加了 `NEXT_PUBLIC_` 前缀，值会被打进前端 bundle，所以 AI API Key 不能使用 `NEXT_PUBLIC_`。:contentReference[oaicite:1]{index=1}  
Vercel 支持在项目环境变量里保存敏感 token，并在构建或函数运行时读取。:contentReference[oaicite:2]{index=2}

9. AI 增强模式
9.1 API 路由

创建：

app/api/analyze/route.ts

功能：

接收 rawIdea
接收当前表单状态
接收模式：overwrite 或 fill-empty
调用 AI 模型进行字段拆解
返回结构化字段

请求体：

{
  rawIdea: string;
  currentForm: PromptFormState;
  mode: "overwrite" | "fill-empty";
}

返回：

{
  ok: boolean;
  data?: Partial<PromptFormState>;
  error?: string;
}
9.2 API Key 安全要求
API Key 存在服务端环境变量，例如：
OPENAI_API_KEY=
不允许使用：
NEXT_PUBLIC_OPENAI_API_KEY
不允许在前端组件中直接读取 API Key。
前端只能请求 /api/analyze。
app/api/analyze/route.ts 在服务端读取：
process.env.OPENAI_API_KEY

Next.js 非 NEXT_PUBLIC_ 环境变量默认只在服务端可用，所以这类 Key 应只在 Route Handler 中读取。

9.3 AI 调用失败处理

如果：

API Key 未配置
请求失败
返回格式不合法
模型超时

则自动降级到离线规则模式，并给出 toast 提示：

AI 增强暂不可用，已使用离线规则拆解。

环境变量不要写进 Git。
提供 .env.example，但不要提交真实 .env.local。

.env.example：

OPENAI_API_KEY=your_api_key_here

Vercel 的环境变量是在源代码外配置的 key-value，可用于构建或函数执行。
