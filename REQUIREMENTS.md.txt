# 项目要求：日常修图关键词完善器静态版

## 1. 项目目标

将用户输入的一句粗略修图想法，整理成可复制、可导出的修图提示词。

项目用途是日常修图、生图、照片风格化提示词整理；不上传图片、不处理真实图片、不生成图片。

## 2. 当前产品方向

- 纯静态 GitHub Pages 页面。
- 只使用浏览器端离线规则。

## 3. 技术栈

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui 风格组件
- GitHub Pages 静态部署

## 4. 核心功能

用户输入 `rawIdea` 后，系统可以：

1. 根据离线关键词规则识别修图方向。
2. 在“覆盖填充”模式下重置为当前识别/选中分类模板，并写入原始想法。
3. 在“只补空项”模式下只填充空字段或仍为默认值的字段。
4. 生成四类输出：
   - 完整提示词
   - 分层关键词
   - 一句话压缩版
   - JSON 结构化提示词
5. 支持复制当前结果、复制全部结果、导出 `.txt`。
6. 支持 localStorage 保存和刷新恢复。

## 5. 分类

至少包含：

- 人物照片轻修
- 生活场景增强
- 胶片 / CCD / 氛围感
- 照片手账注解
- 风景旅行
- 宠物 / 小物件
- 背景清理 / 去杂物
- 通用修图增强

点击分类时，可以补充推荐字段，但不应误删用户已经手动填写的内容。

## 6. 输出要求

所有输出都必须包含底部安全留白规则：每张图底部额外多留不少于 240px 的纯白范围，或约占画面高度 12% 的底部安全留白区，用作后期放文字、排版或避免重要内容被平台信息遮挡；不要在这一区域放主体、注解、装饰元素或杂乱背景。

## 7. 验收标准

- 页面可以输入粗略想法。
- 离线规则可以自动拆解字段。
- 已预选的修图方向不会因为识别不到关键词而自动回到通用分类。
- 点击“智能拆解并覆盖填充”会把原始想法写入字段和生成结果。
- 点击“只补空项”只补齐空字段，不覆盖用户手动填写的有效内容。
- 可以生成完整提示词、分层关键词、一句话压缩版、JSON。
- 支持复制和导出 txt。
- 刷新页面后能恢复上次填写内容。
- 手机端布局正常。
- 可以部署到 GitHub Pages。

## 8. 不要做

- 不要做图片上传。
- 不要真的生成图片。
- 不要引入复杂数据库。
- 不要加登录系统。
